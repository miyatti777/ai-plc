## 実行パターン

### Pattern A: Simple（SNS投稿・定型メール・社内メモ）

```javascript
Role: content_strategist
Skip: Inceptionスキップ
Flow: Collection（テーマ確認）→ Operation（即執筆）
検証: L1のみ（事実確認）
```

### Pattern B: Standard（ブログ記事・プレゼン・提案資料）

```javascript
Role: content_strategist
Flow: Collection（リサーチ）→ Inception（構成案）→ Construction（執筆）→ Operation（レビュー+公開）
検証: L1+L2（事実+可読性+流れ）
```

### Pattern C: Complex（書籍・ホワイトペーパー・シリーズ物）

```javascript
Role: content_strategist
Flow: フル4ステージ（NFR + Extension含む）
SubLayers: チャプター別 / コンテンツ種別別 / フェーズ別
検証: L1+L2+L3（ターゲット読者テスト含む）
Extension: brand-guide（外部公開時）
```

## 検証ステップ（ロール固有 — RUL_plc_system §18拡張）

| Level | 検証項目 | 確認観点 |
| --- | --- | --- |
| L1 | 事実確認 | 引用・数値・固有名詞が正確か |
| L1 | 文法・表記 | 誤字脱字・表記揺れ・文法エラーがないか |
| L2 | 可読性チェック | 流れが自然で、導入→本論→結論が明確か |
| L2 | トーン一貫性 | 文体・世界観が全編通じて統一されているか |
| L3 | ターゲット読者テスト | 想定読者が最後まで読みたくなるか |
| L3 | CTA確認 | 読了後のアクション（申込・共有・購入等）が明確か |

## SubLayer / Task判定

### SubLayer化を推奨

- 異なるコンテンツ種別（ブログ/ホワイトペーパー/動画）
- 大規模コンテンツ群（シリーズ記事、カテゴリ横断企画）
- 段階的な展開（基礎構築 → 本格運用）

### Task化を推奨

- 企画系（テーマ選定、構成案、キーワードリサーチ）
- 制作系（単体コンテンツ: 1記事、1動画）
- 配信系（CMS設定、SNS投稿）
- 分析系（月次レポート、改善提案）

### 注意点

- 品質 vs スピード: 継続性とクオリティのバランス
- 更新・メンテナンス: 古いコンテンツの更新計画
- 再利用可能性: コンテンツ素材の再利用前提

## 汎用NFRチェックリスト（RUL_plc_system §19拡張）

> 📋 **content_strategistロール固有のNFR。**共通4項目（§19）に加え、以下を検証する。
